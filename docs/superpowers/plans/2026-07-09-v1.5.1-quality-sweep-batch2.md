# AnchorScan v1.5.1 Quality Sweep Batch 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove duplicated CLI/Web config helper logic without changing scan behavior.

**Architecture:** Keep the shared behavior in the packages that already own it: `internal/ports` resolves port presets relative to a config file, and `internal/config` loads rule files relative to a config file. CLI and Web call those helpers instead of carrying local copies.

**Tech Stack:** Go 1.26, standard library tests, existing config/ports packages

## Global Constraints

- Keep the change limited to duplicated helper logic.
- Preserve fallback behavior: try files next to the selected config first, then fall back to the repository `config/` directory.
- Verify with `go test ./internal/ports ./internal/config ./cmd/anchorscan ./internal/web` and `make test`.
