# AnchorScan v1.5.1 Quality Sweep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove low-value tracked artifacts, align the repository with the released `v1.5.1` baseline, and tighten the default verification path before the next feature cycle.

**Architecture:** This sweep stays intentionally small. We only touch repository hygiene, version/documentation consistency, test placement, and the default test command; we do not restructure the scan pipeline or change runtime behavior beyond making verification stricter.

**Tech Stack:** Go 1.26, Node.js built-in test runner, Make, Markdown docs, embedded Go templates

## Global Constraints

- Keep the diff minimal and boring; no speculative refactors.
- Do not touch user-local workflow files under untracked `.agents/`, `.codex/`, `.comet/`, `openspec/`, or `skills-lock.json`.
- Verify with fresh commands after implementation: `go test ./...`, `node --test internal/web/static/app.test.mjs`, `make test`, `make package`, `make e2e`.
- Follow the current release line: version text must say `1.5.1` / `v1.5.1`.
- Prefer deletion over archiving when a tracked file has no runtime or test value.
