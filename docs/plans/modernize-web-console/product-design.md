# 现代化 Web Console 产品设计

## 设计北极星

AnchorScan 应表现为一款安静、精确、可信赖的桌面安全工具：第一眼现代简洁，长时间使用不会疲劳，复杂报告仍能快速扫描和操作。

本设计借鉴 Apple 产品界面的克制层次和材质感，但不复制营销页。安全数据、风险状态和操作效率始终高于装饰。

## 视觉原则

### 1. 安静的画布

- 页面底色使用暖白或浅雾灰，不使用纯白铺满所有层级。
- 内容面大多为实色白，靠细分隔线、轻微色差和极浅阴影建立层次。
- 不使用发光、霓虹、厚重渐变、强纹理或大面积品牌色。

### 2. 精确的层级

- 每个页面只保留一个明确主标题和一个主操作。
- 说明文字、元数据、状态和辅助动作按重要性逐级减弱。
- 避免所有内容都包成同等权重的卡片；表格、列表、分组和空白应共同表达结构。

### 3. 轻盈但不松散

- 页面区块之间使用充足留白，控件内部保持紧凑。
- 表格行、筛选器、标签和按钮按专业工具密度设计，不采用营销页尺寸。
- 长报告优先减少视觉噪声，而不是减少可见数据。

### 4. 克制的材质

- 左侧导航、吸顶工具栏和浮层可使用浅色半透明与轻微背景模糊。
- 表格、代码、证据和长文本区域使用稳定实色背景，保证清晰度。
- 圆角、阴影和动画只表达层级或状态，不作为装饰重复出现。

## 初始视觉令牌

以下值已由 ticket 01 原型批准，成为后续页面的统一基准。

| 角色 | 初始方向 |
|---|---|
| Canvas | `#f5f5f7` |
| Surface | `#ffffff` |
| Raised surface | `rgba(255, 255, 255, 0.82)` |
| Primary text | `#1d1d1f` |
| Secondary text | `#6e6e73` |
| Separator | `rgba(60, 60, 67, 0.18)` |
| Accent | `#0071e3` |
| Accent hover | `#0077ed` |
| Focus ring | 蓝色半透明双层焦点环 |
| Radius | 8px / 10px / 14px 三档 |
| Shadow | 仅浮层和提升面使用柔和短阴影 |

风险颜色保留明确语义：critical 红、high 橙、medium 黄、low 蓝、info 灰。风险色不用于普通主按钮或导航选中状态。

字体使用本机系统字体栈。数字、Run ID、端口、命令和证据使用系统等宽字体。正文保持正常字重，避免全局粗体和大写字母造成工业感。

## 已批准的报告构图

- 采用原型 A 的连续工作台：报告标题、运行摘要、风险概览、筛选工作栏和结果表格按单一阅读流排列。
- 从原型 C 保留风险优先级摘要：以简短的“建议先处理”内容突出最高影响发现，不把调查页改成双栏案件档案。
- 不采用原型 B 的固定案件侧栏，避免在 1280px 至 1440px 的工作区挤压结果表格。

## 桌面布局

- 主设计视口：1440px。
- 最低支持宽度：1280px。
- 小于 1280px：允许页面水平滚动，不提供移动导航或重新排版。
- 左侧导航保持固定，目标宽度约 216-232px；主内容根据页面类型使用舒适宽度或数据全宽。
- 仪表盘、表单页使用受控内容宽度；报告和大型表格允许占用剩余工作区。

## 应用外壳

### 左侧导航

- 保留所有现有入口和顺序，允许按“工作流 / 资源 / 系统”进行轻量分组。
- 盾牌 Logo 简化为单色蓝色线性符号，不重新设计品牌字标。
- 当前页面使用浅蓝背景、深色文字和蓝色图标，不使用粗橙色边线。
- 导航项保持紧凑，图标大小和线宽统一。
- 删除移动端汉堡菜单、遮罩和抽屉行为。

### 页面标题区

- 标题、说明、状态和主操作形成稳定横向结构。
- 页面标题不使用全大写 eyebrow 作为强视觉元素；仅在确有分类价值时保留低对比小标签。
- 次要操作进入轻量按钮组或更多菜单，避免标题区堆满同权重按钮。

## 扫描报告页

报告页是首要设计对象，也是后续页面共享语言的验证场。

### 信息结构

1. Run 标识、状态和关键元数据。
2. 主要报告操作：返回、导出、复制或检测命令。
3. 风险摘要与分布。
4. 吸顶筛选工作栏。
5. 视图切换：端口、主机、漏洞。
6. 当前视图结果及分页。

### 筛选工作栏

- 主搜索框始终可见。
- 风险、端口/服务、来源和视图等次要筛选使用紧凑 popover。
- 当前筛选显示为可移除标签，明确区分“已应用”与“可编辑”。
- 重置动作保持可见但弱于“应用筛选”。
- 工作栏可吸顶，但不能遮挡表头或造成双重固定层级混乱。
- 移除 emoji，统一使用现有 SVG 线性图标或纯文本标签。

### 表格和结果

- 表头可固定，列名和排序/分页控件保持低噪声。
- 主识别字段使用较高对比度；协议、来源和元数据降级显示。
- 风险使用紧凑语义标签，不把整行染成风险颜色。
- 证据和长文本默认收起，通过清晰的 disclosure 展开。
- 代码、命令和原始输出使用实色浅灰代码面，支持选择和复制。
- 分页和每页数量形成一个紧凑控制组，避免散落在多个位置。

### 漏洞聚合视图

- 漏洞标题、风险、受影响资产数和可用检测命令形成摘要行。
- 简介、资产和修复建议按阅读顺序分区，不重复相同卡片外观。
- “复制字段”“复制本条”“生成命令”明确区分主次。
- 未匹配、歧义和知识库降级保持现有事实与提示语义，只更新呈现。

## 其他页面

### 仪表盘

- 用一条清晰的运行概览代替四张同权重统计卡堆叠。
- 项目和最近任务是主要内容，系统状态作为次要状态展示。
- 不增加新统计或图表。

### 项目与扫描

- 项目列表突出名称、扫描范围、默认档位和最近状态。
- 项目详情按“概览 / 扫描配置 / 历史任务”组织信息。
- 扫描表单使用清晰字段分组、辅助说明、危险操作提示和固定主操作区。
- 端口、排除项和原始参数保持技术可读性，不隐藏现有能力。

### 扫描历史与任务详情

- 状态、时间、项目和 Run ID 建立稳定列层级。
- 运行中状态可有克制动画，并尊重 `prefers-reduced-motion`。
- 日志和事件使用高可读性等宽区域，不使用终端霓虹效果。

### 单工具、导入与配置

- 表单使用统一标签、帮助文字、错误和成功状态。
- 高级选项通过原生 disclosure 分组，不制造多层嵌套卡片。
- YAML 和命令编辑区域使用稳定代码面，主操作明确。

### 知识库

- 列表强调漏洞名、风险和匹配标识。
- 详情页按说明、匹配、命令和修复信息形成阅读章节。
- 长文本保持适宜行宽，代码和命令独立显示。

### 独立导出报告

- 保持单文件、自包含、可打印和离线可读。
- 使用同一浅色令牌、字体、风险标签和表格语言。
- 不复制 Web Console 的侧边栏、吸顶操作或交互浮层。

## 状态与无障碍

- 所有按钮、链接、输入和 disclosure 提供清晰 `focus-visible`。
- 禁用状态不能只降低透明度到不可读；加载状态保留控件尺寸。
- 成功、警告、错误和风险同时使用文字或图标，不只依赖颜色。
- 交互目标保持桌面可操作尺寸，标签与控件建立语义关联。
- 动画短而克制；对 `prefers-reduced-motion` 关闭非必要动画。

## 截图审批矩阵

每阶段至少提供对应页面的 1440px 截图：

1. 报告页视觉原型：首屏、筛选工作栏、表格片段。
2. 应用外壳：仪表盘与左侧导航。
3. 完整报告：端口视图、主机视图、漏洞聚合视图、展开证据状态。
4. 项目与扫描：项目列表、项目详情、扫描表单。
5. 其余流程：任务详情、工具、导入、知识库、配置。
6. 独立导出报告：屏幕阅读和打印预览。
