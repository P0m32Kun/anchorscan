# Web 控制台 UI 美化设计方案 (Product Design)

**Status:** completed

本方案旨在为 AnchorScan 提供一套兼具现代化审美和专业安全质感的 Web 控制台美化重构设计。本设计参考了市面上优秀的 SaaS 界面（如 Linear、Vercel、Stripe）以及专业的漏洞管理系统（如 Snyk、Nessus），致力于降低高密度安全数据的视觉疲劳，提升操作效率。

---

## 1. 核心设计原则

1. **安静的画布 (Silent Canvas)**：
   - 浅色模式采用雾灰色（`#f5f5f7`）作为背景，面板卡片采用纯白，配合极细描边（`rgba(60,60,67,0.12)`）与柔和短阴影，建立清晰的悬浮层次。
   - 深色模式使用深石墨色（`#0b0b0d`）画布，搭配带有半透明光晕效果的卡片面板（`#1c1c1e`），避免刺眼的高饱和发光。
   - **移除全局卡片的 hover 位移抖动**，保持静态卡片（.panel）的安静稳定，避免大面积页面模块因鼠标滑过而频繁浮动，仅在主操作按钮和链接等交互终点保留微小动效。
2. **精准的层级 (Precise Hierarchy)**：
   - 每个页面最多包含一个主标题和一个主操作（Primary CTA），其余辅助操作使用次级灰色按钮组。
   - 文字粗细（font-weight）和颜色深浅（Primary/Secondary/Muted）严格区分，避免满屏皆粗体或无焦点。
3. **优雅的交互 (Fluid Interaction)**：
   - 摒弃大面积表单和表格的直接平铺，引入滑块 Tab 局部过滤和悬浮目录，实现渐进式披露。
   - 引入**极简单按钮主题控制器**（太阳/月亮图标），一键在明暗偏好间平滑翻转，替代三态冗余文本选择。

## 2. 页面美化重构详解

### 2.1 全局设置页面 (`config.html`)

#### 现状痛点
- 超时配置、引擎路径、高危端口等各种表单垂直平铺在一个长页面，用户修改时需要频繁上下滚动。
- 超时设置有 6 个连续输入框垂直堆叠，占位大且不易对比。
- YAML 原始编辑器和高危端口输入框只是一个原生的 textarea，缺乏安全工具的技术感。

#### 美化方案（Settings Layout）
- **左侧局部导航目录 (Anchor Menu)**：
  - 在页面左侧常驻一个悬浮的局部导航，列出：`外观主题`、`引擎路径`、`超时设置`、`高级 YAML`、`端口预设`。
  - 点击菜单项可实现平滑滚动定位，并带有 IntersectionObserver 的滚动高亮提示。
- **超时设置 Grid 化**：
  - 将 6 个超时输入框调整为 **3x2 的紧凑网格（Grid）**。背景使用极淡的底色（浅色 `#f2f2f7` / 深色 `#1a1a1e`）包围，形成一个逻辑内聚的“超时面板”。
- **IDE 风格编辑器**：
  - 将 YAML 配置和高危端口的输入框改造为 VS Code 风格的深色代码视窗（亮色/深色主题下均为深色底），使用既有等宽字体栈（`JetBrains Mono` 为栈首候选，缺失时回退 `Cascadia Code`/`SFMono-Regular`/系统等宽字体，**不加载任何外部字体文件或 CDN**），设置细腻的代码边界。

#### 页面结构草图
```
+--------------------------------------------------------+
| 配置管理                                   [ 返回面板 ] |
| 全局扫描器配置                                          |
+--------------------------------------------------------+
| [ 外观主题 ]  |  [ 外观：控制台主题卡片 ]              |
| [ 引擎路径 ]  |  =====================================  |
| [ 超时设置 ]  |  [ 基础设置：安全引擎路径表单 ]        |
| [ 高级 YAML]  |  [ 超时设置：(Rustscan | Nmap | Httpx) ]  |
| [ 端口预设 ]  |              (NSE      | Nuclei| rdps)  |
|               |  =====================================  |
|               |  [ 高级 YAML：IDE 风格代码编辑区 ]      |
+---------------+----------------------------------------+
```

### 2.2 控制面板 (`home.html`)

#### 现状痛点
- 整体界面极为平淡，就是一个概览文字加上并排的两张列表卡片，没有任何数据可视化或状态提示。
- 系统状态“运行中”仅为普通文本，缺乏即时就绪的物理安全感。

#### 美化方案
- **就绪状态呼吸灯 (Status Beacon)**：
  - 为“系统状态：运行中”设计一个**带呼吸微动效的绿色物理指示灯**。使用 `::before` 伪元素配合 box-shadow 实现由内向外的绿光晕染扩散；动效强度克制（低透明度、慢周期），且必须通过 `prefers-reduced-motion` 护栏在减弱动效模式下静止。
- **指标卡片化 (Metrics Visuals)**：
  - 将项目库数量、历史任务数量重新包装为现代 Metrics Cards，加入背景色微渐变，以加大字号的等宽数字突显，并增加精细的 CSS 柱状图或迷你进度条装饰。
- **项目与历史列表微动效**：
  - 卡片面板使用高光毛玻璃效果，列表项 hover 时背景色温和渐变，并向右轻微平移 `4px`，引导鼠标焦点。

### 2.3 项目扫描历史 & 网络分区页面 (`project_detail.html`)

#### 现状痛点
- **混乱之源**：目前的扫描历史是按 Zone 循环，Zone 下有完整的 Table。当一个项目拥有 I区、II区、III区等多个分区，每个分区又有十几次扫描时，整个页面会有多张巨型表格，滚动极长、信息完全失控。
- 表格列太宽，如果 IP 或目标资产极长，会将表格撑高变形。
- 网络分区管理只是个简单的列表，操作按钮突兀。

#### 美化方案
- **Zone Tabs 局部过滤（核心改动，已定为唯一方案，不采用 Accordion）**：
  - 彻底改变“所有表格平铺”的现状！在“项目扫描历史”上方引入一个**分区切换按钮组**（如：全部分区、I区、II区、III区、DMZ等，使用 `aria-pressed` 的 toggle button，天然键盘可达）。
  - 点击某个 Zone 时单屏只渲染该分区下的任务表格；默认“全部分区”保持现状平铺，避免隐藏数据造成惊吓。页面垂直长度在用户主动过滤后显著缩短（以“单屏只呈现一个 Zone 的表格”为验收口径，不设拍脑袋百分比）。
  - 纯 vanilla JS 实现，初始化逻辑挂在 `app.js` 既有的唯一 `DOMContentLoaded` 回调内（`app.test.mjs` 有硬断言）。
- **高集成度表格列重构**：
  - **合并 ID 与 Label**：维持现状（Run ID 高亮、Label 小字暗色折行展示），不再改动。
  - **IP 截断与悬浮提示**：目标资产列维持现有 CSS 截断（`max-width` + `text-overflow: ellipsis`）与原生 `title` 悬浮提示，**不新增 `+N` 胶囊与 Popover 组件**（简化决策：原生提示已覆盖查阅场景，避免引入新的定位/遮挡风险）。
  - ~~**工具微标化**~~：**移出本期范围**——表格中展示扫描工具图标需要后端在 Run 视图模型中补充工具数据，超出“不改后端”的约束，待后续单独立项。
- **网络分区管理胶囊流 (Zone Pills)**：
  - 将分区管理重构为**气泡胶囊流**（纯 CSS 改造）。已存在的 Zone 渲染为 Pill，删除按钮缩小内联在 Pill 右侧（**保留“删除”文案与 `.project-zone-item` 结构**，冒烟测试 `web-smoke.mjs` 依赖）。新增分区表单维持极简的 Inline 输入框紧随其后。

#### 页面结构草图

```
+--------------------------------------------------------+
| 项目库 / [项目名称]                       [ 新建扫描 ] |
+--------------------------------------------------------+
| 报告元数据卡片 (被测单位 | 测试对象 | 测试周期)         |
+--------------------------------------------------------+
| 网络分区管理:  [ I区 x ] [ II区 x ] [ III区 x ] [ +新增 ] |
+--------------------------------------------------------+

| |----------------------------------------------------| |

+--------------------------------------------------------+
```

### 2.4 扫描报告页面 (`report.html`)

#### 现状痛点
- 报告信息量极大。缺乏大纲，用户在查找“脆弱性Findings”或“测绘服务”时需要手动滚动很久。
- 漏洞 RAW OUTPUT (证据报文) 已有深色底与语法高亮，但深色模式下盒体与画布底色几乎相同（`#1e1e1e` vs `#1c1c1e`），边界层次丢失。

> **现状校正**：报告**已有** `report-ui.js` 渲染的漏洞占比堆叠图（`#distribution-bar`）和证据盒一键复制按钮（`.evidence-copy-btn`），本节为视觉升级而非从零新增。

#### 美化方案
- **报告悬浮大纲目录 (Floating Outline)**：
  - 页面右侧引入一个常驻的目录组件（**sticky 布局列，不用 fixed 定位**，从机制上杜绝遮挡输入区/表头），实时跟踪并高亮当前所在章节。大纲条目映射模板真实章节（实施时给各 section 补 `id` 锚点）：
    - `风险优先级摘要` → 风险汇总区
    - `检测覆盖事实` → 检测执行覆盖区（无数据时不渲染该条目）
    - `资产与服务测绘` / `漏洞交付聚合` → 依 `AssetView` 二选一
    - `安全脆弱性 Findings` → Findings 表格区（仅测绘视图）
- **漏洞风险堆叠图升级**：
  - 保留现有 `report-ui.js` 统计渲染机制（按当前筛选范围计数），升级视觉：圆角堆叠条、统一使用 severity 设计令牌、图例排版精细化。
- **VS Code 终端风格漏洞证据盒 (Black Terminal Box)**：
  - 维持现有深色终端底色与语法高亮，补齐**深色模式下的描边/阴影分层**，避免盒体与画布融为一体。
  - ~~左侧行号线~~：**不做**（证据为整段 pre 文本，行号需 JS 逐行包裹，收益不抵复杂度）。
