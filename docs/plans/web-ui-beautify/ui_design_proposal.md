## 🛠 五大核心页面美化方案细则

### 1. 全局设置页面 (`config.html`)
> [!TIP]
> 解决痛点：设置项多而杂，长表单滚动繁琐，输入框堆叠无逻辑。

- **两栏锚点导航 (Settings Tabs)**：将长页面改写为“左栏局部导航目录 + 右栏设置卡片”结构。左侧列出“外观”、“引擎路径”、“超时设置”、“高级 YAML”、“端口预设”，点击平滑滚动并自动激活高亮，使复杂设置井井有条。
- **超时网格化 (Timeout Grid)**：将 6 个超时输入框（Rustscan、Nmap 等）从长条垂直堆叠改为 **3x2 的逻辑网格**，搭配浅灰圆角底盒，使时间对比一目了然。
- **IDE 风格编辑器**：YAML 高级编排和高危端口的文本框改造为黑色代码编辑区样式（VS Code 风格），使用 `JetBrains Mono` 等宽字体与细腻的焦点边缘光晕。

### 2. 控制面板页面 (`home.html`)
> [!NOTE]
> 解决痛点：目前控制面板平淡无奇，缺乏视觉焦点和状态提示。

- **就绪状态物理呼吸灯 (Status Beacon)**：在“系统状态：运行中”字样旁设计一个带有呼吸微动效的绿色物理指示灯。通过 CSS 动画由内向外匀速扩散三层半透明绿色光晕，建立高度可靠的就绪感。
- **指标可视化卡片 (Metrics Cards)**：重新设计“项目库”和“历史任务”统计数字卡片，使用特大等宽字体数字，并在下方融入微弱的 CSS 迷你柱状进度装饰。
- **毛玻璃动态卡片 (Glassmorphism List)**：项目列表与最近扫描列表升级为悬浮卡片。列表项在鼠标 hover 时背景色微变并向右平移 `4px`（带 `cubic-bezier` 平滑过渡），建立极佳的动态交互体验。

### 3. 项目扫描历史 & 网络分区页面 (`project_detail.html`)
> [!IMPORTANT]
> 解决痛点：多个 Zone 下的任务表格纵向无脑平铺，Zone 很多时页面长到失控；表格字段极多且挤占空间。

- **Zone Tabs 局部过滤（核心排版优化）**：
  - 彻底改变“所有表格一股脑平铺”的现状！在扫描历史上方渲染一个精致的 **Pills 选项卡导航栏**（如：[全部分区] [I区] [II区] [III区]）。
  - 点击对应 Zone 仅展示该分区下的任务表格，页面垂直长度瞬间减少 70% 以上，让用户只聚焦当前所需。（注：明确排除复杂的 Accordion 折叠设计以保证界面紧凑度）。
- **高集成度表格列**：
  - 合并 Run ID 和 任务 Label 为一列展示（双行排版，ID 高亮，Label 灰色缩进）。
  - 目标资产若是长字符串（如十几个 IP），默认折叠展示为 `+N 个目标` 胶囊，悬浮时再以 Popover 浮层完整展示，杜绝表格行高参差不齐。
  - 增加扫描工具图标组合（微缩 Logo），直观反映本次扫描运行了哪些安全探针。
- **网络分区胶囊流 (Zone Pills)**：
  - 将分区管理区域重新整合成 Inline 胶囊流。已创建的分区以灰度圆角 Pill 展示，右侧有隐藏的删除按钮；新增分区表单做成极简的 inline Input 直接尾随其后。

### 4. 扫描报告页面 (`report.html`)
> [!TIP]
> 解决痛点：报告信息量极大，纵向滚动易迷失；漏洞分布展示抽象；证据报文无层次。

- **大纲目录悬浮导航 (Report Outline)**：页面侧边常驻一个悬浮的大纲组件（`1. 风险优先级 / 2. 检测覆盖 / 3. 资产测绘 / 4. 漏洞 Findings`），随页面滚动动态激活高亮，一键点击平滑锚点跳转。
- **漏洞风险堆叠彩虹图 (CSS Stacked Chart)**：使用纯 CSS 设计一个横向百分比堆叠图（Critical 绯红 / High 橙黄 / Medium 琥珀 / Low 蓝色），让风险比例一眼即知。
- **VS Code 终端风格证据盒 (Technical Terminal)**：
  - 漏洞详情中展开的 RAW 报文/响应，背景采用纯黑终端色（`#0b0b0d`）。
  - 使用绿色/青色等宽字体，搭配精致的左侧行号和“一键复制数据”按钮，模拟真实终端取证质感。
