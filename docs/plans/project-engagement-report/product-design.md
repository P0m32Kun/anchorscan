# 项目级渗透测试与报告产品设计

## 信息架构

### Project 列表与创建

Project 卡片表达一次测试任务，而不是一段网段：显示任务名称、被测单位、日期、Zone 数量、Run 数量、待确认漏洞数量和报告完整度。

创建页只采集任务信息：任务名称、被测单位、报告标题、日期、测试人员和说明。报告元数据（报告标题、测试对象、测试人员、起止日期等）只在项目管理界面（创建/编辑 Project）维护，报告页不再另设元数据表单。创建后默认生成 I区、II区、III区，用户可以新增“其他” Zone；不在此处出现目标、端口、排除项和扫描档位。

### Project 详情

固定四个入口：

1. **概览**：任务元数据、Zone、Run/验证/证据数量和报告完整度；
2. **扫描任务**：按 Zone 分组的 Runs，提供新建、重新运行和是否纳入报告；
3. **验证工作台**：处理正向候选、负向候选和检查未完成项；
4. **正式报告**：预览结构、缺失项检查和下载 HTML / DOCX。

## 发起扫描

表单首屏直接显示本次扫描必填项，不再藏在“覆盖项目默认值”折叠区：

- Zone（必填，默认选择用户进入时所在 Zone）；
- 扫描名称（可选，空值自动生成“Zone + 时间”）；
- 目标（必填，支持现有 IP/CIDR/范围/文件导入）；
- 端口（必填或使用清晰显示的全局默认）；
- 档位（默认全局 `normal`，但保存到 Run 快照）；
- 排除目标、排除端口；
- 接入点/交换机说明、测试机 IP、备注；
- 高级引擎参数。

重新运行复用原 Run 的 Zone 和全部表单值，但仍创建新 Run，并要求用户确认接入点和目标没有变化。

## 验证工作台

### 全局过滤

顶部始终保留 Project、Zone、来源 Run、漏洞等级、服务、端口、验证状态和关键词。过滤结果作用于候选列表、复制资产和命令生成，但不隐式改变“纳入报告”状态。

### 待确认漏洞

卡片按漏洞而不是 Finding 展示：

- 正式标题和危险等级；
- 跨 Runs 聚合后的唯一 IP:port 数量与列表；
- 来源 Zone/Run；
- 知识库简介和修复建议；
- 复制 IP:port、复制整条；
- 生成批量 Nuclei/Nmap/MSF 命令；
- “开始验证”或已有 Verification 状态；
- Evidence 缩略图数量、粘贴/上传入口；
- “确认存在并纳入报告”。

创建 Verification 时复制知识库当前标题、等级、简介和修复建议作为报告草稿，允许人工修订；后续知识库变化不应静默改写已经确认的正式交付内容。

### 待负向验证

按指纹端点展示服务、产品、版本、NSE/nuclei 完成状态和 `info` 输出摘要。这里只表示“具备人工复核条件”，不显示“安全”或“不存在漏洞”。

用户必须：选择一个知识库验证项或手工命名验证项 → 勾选本次验证覆盖的同类候选端点（可全选） → 生成/复制命令 → 执行验证 → 粘贴/上传一张共享截图 → 提交“本次验证未发现”，生成一条关联全部选中端点的 Verification。同一类验证项只需一张截图，不要求每个 IP 单独截图。缺少任一步时保持草稿或无法判定。

### 检查未完成

集中显示 DetectionCheck 缺失、失败、跳过、取消和中断的端点，并提供重新运行相关工具的入口。旧 Run 缺少 DetectionCheck 时明确显示“历史执行事实不可用”，不能与双引擎零发现混为一谈。

## 工具验证闭环

现有报告页已经能生成命令和以 `raw_args` 打开工具页；新流程扩展上下文参数：

- `project_id`；
- `zone_id`；
- `verification_id`；
- `raw_args`；
- `return_to`。

工具页自动选择 Project/Zone，显示当前验证项摘要和“返回验证项”。工具运行创建 `kind=tool` 的 Run，默认不直接纳入正式报告。完成页提供：返回验证项、复制输出、粘贴/上传截图。截图仍由用户决定是否构成 Evidence，系统不自动截屏或分析图片。

## Evidence 交互

验证卡片提供一个原生上传区：

- 点击选择文件；
- 拖拽 PNG/JPEG；
- 页面聚焦时使用剪贴板粘贴截图；
- 上传后立即显示缩略图；
- 可填写说明、拖动排序和删除；
- 明确显示“第 N 张，将按此顺序进入报告”；
- 明确提示“同一类漏洞/验证项只需一份截图，覆盖本条 Verification 的全部关联资产”；覆盖多个资产时，说明必填并写明本次验证范围。

服务端拒绝非图片、伪造 MIME、超限文件和异常尺寸。删除只影响 Verification 的报告证据，不删除工具 Artifact 或历史 Run 事实。

## 正式报告编辑与预览

正式报告页分为“元数据”“汇总表”“按 Zone 详情”“结论”四段，并持续显示缺失项计数。元数据段只读引用 Project 管理界面维护的报告标题、测试对象、测试人员、起止日期；缺失时显示错误并提供跳转回 Project 编辑页的入口。

### XX 占位替换

元数据来自 Project 管理界面：被测单位、报告标题、测试对象、日期、测试人员。接入点、测试机 IP 和范围说明来自各次 Run；同一 Zone 多次更换交换机时按 Run 分行展示，不把不同现场信息压成一个 Zone 字段。结论统计自动计算，结论补充文字允许人工编辑。预览中任何未绑定字段以错误状态显示，下载按钮禁用；最终 HTML 不允许出现模板占位 `XX`。

### 表 3-1

表格实时从已纳入的 `confirmed` Verifications 派生，不提供单独编辑副本。用户修改验证标题、等级、关联资产或纳入状态后，汇总表同步变化：

| 列 | 数据来源 |
|---|---|
| 序号 | 当前稳定排序后的行号 |
| 安全问题 | Verification 正式标题 |
| 关联资产/域名 | 跨 Runs 去重并排序的 IP:port |
| 严重程度 | Verification 最终等级 |

### Zone 详情

固定顺序输出 Project Zones。每个 Zone 先显示范围与接入信息，再按等级和用户排序输出 confirmed 漏洞，最后输出 not_observed 验证项。没有内容的 Zone 显示“本区未纳入报告条目”，不伪造零风险结论。

每个 confirmed 漏洞详情固定按“标题与等级 → 漏洞描述 → 漏洞详情 → 关联资产 → Evidence → 修改建议”输出。每个 not_observed 验证项固定按“验证项 → 关联资产 → 验证说明 → Evidence”输出。截图只按用户顺序展示并使用说明文字，不做 OCR、内容识别或自动判定。

“测试工具与方法”不能照抄 DOCX 中的旧工具清单。默认根据纳入 Runs 的实际配置生成 RustScan、Nmap、httpx、Nuclei 等真实使用项，并允许补充人工工具；没有运行证据的工具不得出现在正式报告中。

## 导出收敛

- Run Report 页头只保留“下载 HTML”。
- 删除“下载 JSON”“下载 CSV”和资产 CSV；保留复制 IP、复制 IP:PORT、复制 URL及必要 TXT 下载。
- Project Report 提供两种下载：单文件 HTML（CSS 和必要资源全部内嵌，离线打开可用）和 DOCX（由干净模板填充生成）。
- 内部 `report.json` 不作为用户下载格式展示。

## 甘肃数据迁移体验

迁移功能不能按名称自动猜测。迁移前备份数据库，由用户新建一个甘肃任务 Project 和 I/II/III Zones，然后明确选择旧 Project 的 Runs 移入：

- 旧“1区-1-2-5”“1区-3”“1区-50” Runs → I区；
- 旧“3区” Run → III区；
- II区保留为空。

迁移预览必须显示 Run 数量、状态、Artifact 目录和将要移动的归属；确认后移动数据库归属与受管理目录，旧 Project 只在为空且用户再次确认时删除。
