# 报告风险等级与可用性修复

**状态：approved（2026-07-22）**

本计划处理用户已确认的报告可见性问题，并以 Nuclei 的 `critical`、`high`、`medium`、`low` 风险等级为统一口径。

## 范围

1. 扫描报告的风险摘要须分别显示严重、高危、中危、低危的实际数量；低危不得因摘要模型遗漏而消失。
2. 项目 DOCX 报告的结论须显示严重漏洞数量；报告模板和知识库使用手册须将“严重”作为可用、可统计等级。
3. 漏洞聚合页面可按漏洞名称或漏洞 ID 过滤，并可复制当前过滤结果的全部 `IP:port`。
4. 级联流水线终端刷新不得破坏用户正在进行的文本选择与复制。

## 非范围

- `internal/tools/httpx.go` 与 `internal/tools/httpx_test.go` 是用户正在处理的 Windows 扫描兼容性修复，本计划不修改、覆盖或提交这两份文件。
- 不新增 Nuclei 以外的严重程度，也不改变已有严重程度的存储值。

## 验收标准

- 一条低危发现会使扫描报告的低危计数为 1，且不影响其它等级计数。
- 含一条严重发现的项目交付模型，会将 `conclusion.critical` 传入 DOCX 渲染上下文；生成的结论文本显示“严重漏洞 1 个”。
- 知识库手册明确列出“严重（critical）”，并与既有解析规则一致。
- 后续工单的漏洞名称/ID 筛选与批量 `IP:port` 复制，仅作用于当前过滤结果；终端轮询在用户选择文字时不重建已显示内容。

## 测试接缝

- 报告 HTTP/view model：风险摘要的用户可见计数。
- DOCX 渲染上下文：交付模型到模板的等级统计契约。
- DOCX 渲染结果：正式模板结论文本和版式不回归。
