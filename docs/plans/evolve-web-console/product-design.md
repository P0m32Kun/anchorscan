# Web Console 持续演进产品设计

**Status:** proposed

## Design Intent

保留已批准的克制、桌面优先、Apple-inspired 视觉方向，不改成“黑客终端”主题。新一轮设计优先减少决策与重复操作，再处理装饰。暗色主题用于长时间和低光环境工作，不采用纯黑、霓虹绿或大面积发光边框。

规范调研与验收依据见 [现有 UI/UX 调研](../modernize-web-console/ui-ux-research.md)。

## Global Theme

- 偏好固定为 `跟随系统 / 浅色 / 深色`，默认跟随系统。
- 侧边栏底部提供一个紧凑的“外观”入口，打开原生 radio 语义的三项选择；切换立即生效且不需要保存。
- 配置页同步显示同一偏好，但不建立第二份状态。
- 表面令牌至少区分 canvas、surface、raised、overlay、separator、text、muted、accent、focus、code 和 risk levels。
- 导航强调色与风险色分离；严重性继续使用“文字 + 色彩/图标”，不能只靠颜色。
- 独立 HTML/DOCX 报告保持浅色打印外观。

## Interaction Principles

1. **主任务先出现**：高频、必填、决定范围的控件始终可见。
2. **只披露一层**：低频选项进入一个高级区域，不嵌套 accordion。
3. **一个明确主操作**：每个卡片或表单最多一个视觉主按钮；命令、复制和危险操作降级或收纳。
4. **反馈不打断**：字段错误就地显示，异步状态使用页面内 status；只有不可逆操作使用确认 dialog。
5. **状态不靠颜色**：Run、Finding、Coverage 均显示完整文字与稳定视觉线索。
6. **鼠标和键盘同权**：popover、tabs、dialog 和 disclosure 有完整焦点进入、移动、Esc 关闭与焦点恢复。
7. **保留上下文**：筛选、轮询和复制反馈不重置滚动、展开项或当前焦点。

## Page Evolution

### Scan Create

- 首屏只保留 Project/Zone 上下文、目标、Profile、端口与“开始扫描”。
- 只有一个 Zone 时自动选中；多个 Zone 时仍要求明确选择。
- 排除目标、排除端口、标签、备注、工具参数和产物路径按风险与频率归入“可选信息”或“高级选项”，最多一层披露。
- 高级项已修改或包含错误时自动展开，并在标题显示修改数量。
- 提交后防止重复点击，创建成功直接进入 Run 详情。

### Verification Workbench

- 搜索与最常用的状态/风险筛选常驻，其余放入“更多筛选”。
- 候选卡片保留一个主操作“验证”；生成命令、复制和资产级操作按上下文展开。
- 页签支持方向键、Home/End，并维护明确的选中与焦点状态。
- 删除证据等破坏性动作继续确认；一般校验与保存结果不再使用 `alert`。

### Report

- 风险摘要、发现和资产结果优先；检测执行事实默认折叠。
- 搜索常驻，低频筛选收纳；活动筛选始终可见并可逐项移除。
- 查询参数仍是筛选结果的可分享来源，不把关键状态只存进 Vue 内存。
- 表头排序、分页和行级动作使用真实按钮或链接，固定层不能遮挡焦点。

### Run Detail

- 显示生命周期状态、当前阶段、可靠计数、最新事件与下一步动作。
- 只有分母可靠时显示百分比；否则使用阶段文字和不确定进度。
- 轮询不刷新整页、不抢焦点、不关闭展开项；用户可以离开后从历史返回。

### Remaining Pages

- 项目、导入、工具、知识库和配置页只迁移真实存在的交互状态；纯展示内容继续由 Go 渲染。
- 不为了“Vue 覆盖率”把静态表格、链接和简单 POST 表单包装成组件。

## Visual Acceptance

每个阶段至少检查 1440px 浅色与深色截图，并补充 1280px 可用性检查。状态矩阵包含：默认、hover、focus-visible、disabled、loading、empty、error、popover/dialog 打开、长文本和高数据密度。
